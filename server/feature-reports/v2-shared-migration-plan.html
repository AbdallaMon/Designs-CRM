Old Code vs New Code Patterns

Old Way (routes/shared)
700-line client-leads.js doing everything
Manual try/catch in every route handler
No input validation — raw req.body
Role checks inline in route (12+ lines each)
getTokenData(req,res) / getCurrentUser(req)
utilities.js = 3 unrelated domains mixed
Duplicate endpoints (/update/:id AND /lead/update/:id)
console.log scattered everywhere
No standard response shape enforcement
New Way (v2/modules)
9 focused sub-modules under leads/staff/
asyncHandler + AppError + global errorHandler
Zod validate() middleware on every route
resolveScope(req.auth) inside each usecase
req.auth set by AuthMiddleware.requireAuth
Split: notifications + user-logs + utilities
Single PUT /:id/update in v2
Errors bubble to global handler silently
ok() / created() / updated() from shared/http

Why client-leads.js Needs to Be Split into 9

The old client-leads.js is 700+ lines with 26 endpoints spanning 6 conceptual domains. In v2 it becomes 9 focused sub-modules all under leads/staff/:

lead-list/
GET / · GET /deals · GET /columns
lead-detail/
GET /:id · PUT /:id/status · PUT /:id/update
lead-assignment/
PUT /assign · PUT /bulk-assign · PUT /convert · POST /:userId/countries
call-reminders/
GET /calls · POST /:id/call-reminders · PUT /call-reminders/:id
meeting-reminders/
GET /meetings · GET /:id/meeting-reminders · GET /meeting-reminders/:id · POST /:id/meeting-reminders · POST /:id/meeting-reminders/token · PUT /meeting-reminders/:id
price-offers/
POST /:id/price-offers · POST /price-offers/change-status
lead-payments/
POST /:id/payments
lead-files-notes/
POST /:id/files · POST /:id/notes · POST /:id/payment-reminder · POST /:id/complete-register
sales-stages/
GET /:clientLeadId/sales-stages · POST /:clientLeadId/sales-stages

Middleware Reference (v2 Pattern)

Middleware Import From Purpose
AuthMiddleware.requireAuth shared/middlewares/auth.middleware.js Sets req.auth from cookie token
AuthMiddleware.requireRole([...]) shared/middlewares/auth.middleware.js Admin-only route guard
asyncHandler(fn) shared/middlewares/async-handler.js Catches async throws, calls next(err)
validate(schema, source) shared/middlewares/validate.middleware.js Zod schema validation
AppError(msg, status) shared/errors/AppError.js Typed error with HTTP status code
ok / created / updated / deleted shared/http/response.js Standard JSON response helpers

Risks & Notes

Risk Mitigation
Frontend calls /shared/... URLs Old routes stay live in parallel. Frontend migrates per-feature.
Complex role logic (9 roles + isSuperSales) Each usecase gets a resolveScope(auth) helper that encapsulates the branch.
Duplicate endpoints /update/:id + /lead/update/:id v2 exposes a single clean PUT /:id/update. Old endpoints untouched.
deleteAModel is generic — deletes any model Add allowed-model validation in tasks usecase.
createFile may need multer upload pipeline Verify if it saves a URL or binary. Add multer middleware if binary upload needed.
getAdminClientLeadDetails vs getClientLeadDetails Branch cleanly in lead-detail.usecase.js based on req.auth.role.
